## Раздел 1 (правила git)
1. Перед работой над новой функцией проекта переключись на основную ветку и получи последние изменения проекта:
```shell
git switch main 
git pull origin main
```
2. **Каждую новую функцию пишем в своей новой ветке** для этого находясь в main ватке пишем команду:
```
git switch -c your-feature-name  
# -c создает и сразу переключает на ветку
```
3. Работаем над функцией. Далее когда есть готовые изменения - коммитим.
```
git add your_files
git commit -m "your comment"
```
4. Пушим ветку на GitHub (в первый раз с -u)
```
git push -u origin your-feature-name
```
5. Для последующих коммиртов в этой ветке:
```
# 1. Добавляем файлы
git add .
или 
git add your_files
# 2. Создаем коммит  
git commit -m "feat: add new improvements"
# 3. Пуш БЕЗ указания ветки!
git push
```
6. Слияние с **main** веткой:
> Делается после утверждение функции командой.
```
# 1. Переключиться на вашу feature-ветку (если ты не в ней)
git switch your-feature-name 
# 2. Получить последние изменения из main
git fetch origin main 
# 3. Добавил все изменения в свою ветку из main
git merge origin/main 
```
> Если возникают конфликты: 
> 1. Открой файлы с конфликтами
> 2. Удали <<<<<<<, =======, >>>>>>>
> 3. Оставь нужный код
> 4. Зафиксируй
```
git add .
git commit -m "fix conflicts"
```
**Завершение слияния:**
```
git switch main
git merge your-feature-name 
git push origin main
```

## Раздел 2 (правила Кода)
### Скрипт
1. Новый скрипт создаётся в папке src/utils; 
2. Имя скрипта должно соответствовать его функционалу, но не должно быть слишком длинным.
3. Каждый скрипт должен начинаться с многострочного комментария, который кратко описывает функционал:
```py
"""
information
"""
def func():
```
4. Один скрипт содержит одну или несколько функций. Если функций несколько - то они должны соответствовать одной логике. 
> Например скрипт file.py содержит в себе функции load_file(), save_file().

### Функции
1. Функция выполняет только одну задачу. 
2. Добавляем комментарии внутрь функции, только если реализована сложная логика.
3. Все переменные должны иметь логичное имя. 
4. Функция должна быть максимально проста и читаема.
